# 先写方案再操作规范

## 核心原则

**任何批量操作或破坏性操作，必须先写方案文件，用户确认后才能执行。**

## 适用场景

| 场景 | 是否需要先写方案 |
|------|-----------------|
| 整理/归档文件 | ✅ 必须 |
| 批量重命名 | ✅ 必须 |
| 删除文件/目录 | ✅ 必须 |
| 目录结构重构 | ✅ 必须 |
| 批量修改配置 | ✅ 必须 |
| 数据迁移 | ✅ 必须 |
| 单个文件编辑 | ❌ 不需要 |
| 新建单个文件 | ❌ 不需要 |
| 读取/查询操作 | ❌ 不需要 |

## 方案文件规范

### 文件命名

```
[操作类型]方案_[日期YYYYMMDD].txt
```

示例：
- `文件整理方案_20260127.txt`
- `目录重构方案_20260127.txt`
- `批量删除方案_20260127.txt`

### 文件位置

- 放在操作涉及的主要目录下

### 必需内容

```
# [操作名称]方案
# 日期：YYYY-MM-DD
# 状态：待审核

## 一、操作范围
[说明本次操作涉及哪些文件/目录]

## 二、操作原则
[说明分类/处理的依据和原则]

## 三、具体计划
[列出每个文件的处理方式：迁移到哪里、为什么]

### 3.1 [分类1]
| 源文件 | 目标位置 | 原因 |
|--------|----------|------|
| xxx    | xxx      | xxx  |

### 3.2 [分类2]
...

## 四、执行步骤
[按顺序列出执行步骤]

## 五、风险提示
[列出可能的风险和注意事项]

## 请确认
请审核以上方案，确认后回复"同意"或提出修改意见。
```

## 工作流程

```
用户请求批量操作
       ↓
  写方案文件（.txt）
       ↓
  告知用户方案位置
       ↓
  等待用户确认
       ↓
 ┌─────┴─────┐
 ↓           ↓
同意      修改意见
 ↓           ↓
执行操作   修改方案
 ↓           ↓
生成报告   重新确认
```

## 禁止事项

- ❌ 未写方案直接执行批量操作
- ❌ 用户未确认就开始执行
- ❌ 方案中遗漏重要信息（如删除原因）
- ❌ 执行时不按方案来

## 示例对话

### 正确示范

```
用户: 把这些文件整理一下

AI: 让我先分析目录结构，然后写一个整理方案。
    [分析目录]
    [写方案文件]
    
    方案已写入 `文件整理方案_20260127.txt`，请审核。
    主要内容：
    - 迁移 XX 个文件到 .oa/docs/
    - 迁移 XX 个文件到业务项目目录
    - 建议删除 XX 个过期文件
    
    确认后我再执行。

用户: 同意

AI: [执行迁移操作]
    [生成完成报告]
```

### 错误示范

```
用户: 把这些文件整理一下

AI: [直接开始移动文件]  ← ❌ 错误！未写方案未确认
```
