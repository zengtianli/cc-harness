# 农业灌溉需水预测模型

> 适用文件: `ngxs/**/*.py`, `灌溉模型B3案例数据/**/*`

## 一、模型概述

### 1.1 模型定位
基于水量平衡原理的农业灌溉需水计算模型，用于预测灌区逐日/逐旬灌溉需水量。
覆盖浙东地区 15 个平原区域，总面积 4600+ 平方公里。

### 1.2 核心公式
```
水位变化 = 降雨 + 灌溉 - 蒸发蒸腾 - 渗漏 - 排水

灌溉决策: if 水位 < H_min -> 灌溉量 = Storage - 当前水位
排水决策: if 水位 > H_max -> 排水量 = 当前水位 - H_max
```

### 1.3 目录结构
```
ngxs/
├── main.py           # 主程序入口
├── calculator.py     # 主计算控制器
├── paddy_models.py   # 水稻灌溉模型
├── dryland_models.py # 旱地作物模型
├── nc_adapter.py     # B3 NC数据适配器
├── config.py         # 全局配置
├── config_b3.py      # B3专用配置
├── utils.py          # 工具函数
├── core.py           # 模拟基类
└── data/             # 输出结果
```

## 二、计算模式

| 模式 | config.py 值 | 输出文件 |
|-----|-------------|---------|
| 旱地作物 | `dryland` / `crop` | OUT_GGXS_C.txt, OUT_PYCS_C.txt |

## 六、比赛要求与现状对比

### 6.1 比赛评测要求
- **测题**: 计算灌区灌溉期**逐旬**需水量
- **评测指标**: 灌溉期总量相对误差、单旬相对误差最大值（误差 <=25% 为合格）
- **输入**: 气压、风速、气温、湿度、日照、降水、灌溉面积、作物结构、水利用系数
- **输出**: 灌溉区逐旬农业灌溉需水量 (NC 格式)

### 6.2 对比分析

| 项目 | 比赛要求 | 当前模型 | 改造状态 |
|-----|---------|---------|---------|
| 输出时间尺度 | 逐旬 | 逐日 | 需加旬聚合 |
| 数据格式 | NC | 已支持 | 已完成 |
| 精度要求 | <=25% | ~21% | 基本达标 |
| 数据年限 | 5年 | 1年(2020) | 需扩展 |
| 作物系数 Kc | 需率定 | 初步设置 | 待优化 |
| 模型封装 | .so/.jar | Python | 需封装 |

### 7.2 Excel vs NC 的关系

Excel 是给人看的原始数据，NC 是给程序用的标准格式，内容相同。

```
Excel (2020逐日数据.xlsx)           NC 文件
----------------------------------------------
降雨量（mm）                   ->   rain_2020.nc
ET0（mm/day）                  ->   et0_2020.nc
气温、风速、湿度、日照...       ->   temp_*.nc, wind_*.nc...
日灌溉水量m3 (实际值)          ->   output_2020.nc (IRRIGATION_WATER)
```

### 7.3 实际数据（标准答案）

`output_YYYY.nc` 里的 `IRRIGATION_WATER` 变量 = 实际灌溉量（真值）

| 2022 | 率定数据/.../output_2022.nc | 114,217 万m3 | 参数率定 |
| 2024 | 率定数据/.../output_2024.nc | 101,545 万m3 | 参数率定 |

### 7.4 数据用途流程

```
第1步: 建模数据 (2020年)
  输入: rain, et0, crops... (NC文件)
  真值: output_2020.nc
  用途: 开发调试模型 (已完成)
        |
        v
第2步: 率定数据 (2021/2022/2024/2025)
  输入: rain, et0, crops... (NC文件)
  真值: output_YYYY.nc
  用途: 调整参数(蒸发系数Kc等)，让模型更准
        |
        v
第3步: 比赛测试 (比赛时给的新数据)
  输入: 给你新的 NC 文件
  真值: 不告诉你，用于评分
  评判: 你输出的 vs 真值，误差 <=25% 合格
```

### 7.5 如何验证模型

```bash

# 3. 读取真值

# 4. 计算误差
# 误差 = |模型输出 - 真值| / 真值 x 100%
# 目标: 总量误差 <=25%, 单旬误差 <=25%
```
