# 2025风险图项目架构

## 项目概览

### 项目类型
- **水利风险分析** + **GIS空间数据处理**

### 涉及流域
- 华溪（HX）、熟溪（SX）、白溪（BX）等

### 主要工作
1. **QGIS空间处理** - 10步工作流
2. **Excel表格填充** - 3套表格（库表结构、预报调度、风险分析）
3. **数据整理** - 多源合并、字段标准化

## 任务分配规则

### 根据任务类型自动分配

| 用户说 | 应调用能力 |
|--------|-----------|
| "处理 GeoJSON" / "导出 Shapefile" / "空间数据" | GIS空间处理 |
| "筛选数据" / "合并表格" / "Excel" | 数据处理 |
| "写脚本" / "有没有现成工具" | 脚本工程 |
| "整理项目" / "归档" / "清理" | 结构管理 |

### 复杂任务：多步协作

**示例：处理新流域数据**
```
用户：帮我处理白溪的数据

流程：
Step 1: GIS空间处理
   - 读取 bx_env.geojson、dd.geojson
   - 导出标准格式的保护片和堤段

Step 2: 数据处理 - 整理Excel数据
   - 从 风险分析-白溪.xlsx 提取参数
   - 生成 economy.txt、LossRate.txt

Step 3: 脚本执行
   - 复制 organize_sx_data.py → organize_bx_data.py
   - 修改配置参数并运行
```

## 项目工作流

### QGIS完整处理流程

```
原始数据
    |
    v
QGIS 10步工作流
  01 生成河道中心点 → 02 切割堤段 → 03 赋值高程
  04 对齐字段 → 05 增强Grid → 06 House层
  07 Road层 → 08 Vegetation层 → 09 对齐输出
  10 保护片层 → 99 批量导出
    |
    v
导出的 GeoJSON/Shapefile
    |
    v
Excel填充
  1.x 库表结构 → 2.x 预报调度 → 3.x 风险分析
    |
    v
最终成果
```

## 新流域完整处理流程

处理新流域前需确认：
1. 流域名称（如：白溪）
2. 流域代码（如：bx）
3. 原始数据位置

处理阶段：

```
Phase 1: 空间数据处理
├─ 处理 GeoJSON/Shapefile
└─ 输出：标准格式的图层文件

Phase 2: Excel数据整理
├─ 提取参数、填充表格
└─ 输出：economy.txt、风险分析表

Phase 3: 脚本执行
├─ 运行 organize_xx_data.py
└─ 输出：完整成果目录
```

每阶段完成后应检查，确认后进入下一阶段。
