description: 单节写作（原文照抄+上角标+4维度自检），输出MD片段
---

用户要写报告的一个小节。

## 输入

用户会在对话中提供：
1. **小节标题**（如 "2.1 项目区概况"）
2. **参考资料**（成品报告片段、数据文件路径、或直接贴文本）

如果缺少以上任一项，问用户要。

## 标题编号体系（7级）

正文中的标题严格按以下层级编号，不得使用"一、二、三"、"（一）（二）"等中文序号：

| 层级 | 格式 | 示例 |
|------|------|------|
| 1级 | `1` | 1 总论 |
| 2级 | `1.1` | 1.1 项目背景 |
| 3级 | `1.1.1` | 1.1.1 自然概况 |
| 4级 | `1.1.1.1` | 1.1.1.1 地形地貌 |
| 5级 | `（1）` | （1）政策解读与任务分解。 |
| 6级 | `1)` | 1）灌区管理单位 |
| 7级 | `①` | ①核对灌溉面积 |

规则：
- 用户给的小节标题决定当前层级，往下的子标题按体系顺次递进
- 每个编号后加一个空格再写标题文字
- 不要用 markdown 的 `#` 标记做标题层级，直接用编号加粗：如 `**1.1 项目背景**`
- 5级（1）开头的标题后面跟正文时，标题加粗后直接接正文（如 `**（1）政策解读与任务分解。**正文...`）

## 写作规则

### 原文优先
- **尽可能从参考资料原文照抄**，保留经过审定的正式表述
- 需要适配当前县市的部分（地名、数据、水库名等）做替换，其余措辞尽量保留原文
- 每处引用加上角标 `[n]`，同一来源用同一编号

### 4维度自检（写的时候就做到，不是写完再查）

1. **完整性** — 不留空、不截断、逻辑链完整，每段有头有尾
2. **结构性** — 先介绍后分析、先现状后方案，段落有起承转合
3. **立场措辞** — 乙方立场，用"建议""宜"，不用"必须""确保"；引用政策注明文号
4. **数据一致性** — 数字、单位前后统一，与给定数据源一致

### 篇幅匹配
- **插入内容的篇幅必须与所在文档中同级兄弟节保持一致**
- 写之前先看目标文档中相邻章节的长度和密度（几个条目、每条几句话），以此为标尺
- 如果兄弟节都是 2-4 条短条目，你也写 2-4 条，不要展开成多级子标题的长文
- 如果兄弟节每节 2000+ 字带多级结构，你也按该密度写
- 篇幅异常（比兄弟节长/短 2 倍以上）= 不合格，必须调整

### 禁止
- 不要加 AI 味的过渡句（"综上所述""总而言之"）
- 不要自己编造数据，数据必须来自用户给的资料
- 不要一次写多节，只写用户要求的那一节

## 输出格式

直接在对话中输出 Markdown，格式：

```
**{编号} {小节标题}**

{正文，含上角标[1][2]...，子标题按7级体系编号}

---
**参考文献**
[1] {文件名/来源} — {页码/章节}
[2] ...
```

## 完成后

告诉用户这一节写好了，问是否需要调整，或者继续写下一节。
