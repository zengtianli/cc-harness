---
description: 合并多个修改稿 DOCX 的修订内容到基准文档（等效于全部"接受修订"后合并）
---

用户提供了参数: $ARGUMENTS

将多个（修改稿）DOCX 的修订内容合并写入基准 DOCX。等效于在每个修改稿中"全部接受修订"后，把各节改动合并到一个文件。

## 参数

- 文件列表：多个修改稿 DOCX 路径，空格分隔
- 无参数：自动扫描 `招标文件/（修改稿）*.docx`，全部合并
- `--base <file>`：指定基准 DOCX（默认自动找 `招标文件/` 下的主文档，排除 ~$、copy、修改稿）
- `--dry-run`：只列出将合并的规则，不写入

## 原理

每个修改稿都有对应的 `招标文件/_revise_rules_{章节号}.json`，里面记录了 find/replace 规则。
合并 = 把所有规则合在一起，对基准 DOCX 做**直接替换**（不加修订标记），输出一个干净的合并稿。

## 流程

### 1. 确定输入

找基准 DOCX 和所有修改稿文件。从修改稿文件名提取章节号，匹配对应的 rules JSON。

### 2. 合并所有规则

```python
all_rules = []
for rules_path in matched_json_paths:
    with open(rules_path, 'r', encoding='utf-8') as f:
        rules = json.load(f)
    all_rules.extend(rules)
```

### 3. 直接替换写入

用 python-docx 对基准 DOCX 逐段落做文本替换（**不加修订标记**，直接替换原文）。

**关键细节**：
- `replace` 为空字符串 → 删除整段（清空段落文本，或移除段落元素）
- `replace` 含 `\n` → 需要拆分为多段落插入
- 替换时保留原段落格式（style、字体等）
- 找不到匹配的规则要警告，不要静默跳过
- **多文件合并按从后往前顺序处理**，避免索引漂移

### 4. 输出与报告

- 输出文件名：`招标文件/标书大纲-合并稿-{日期}.docx`（日期 = MMDD）
- 打印汇总表：

| 规则来源 | 规则数 | 已应用 | 跳过 |
|----------|--------|--------|------|

- 用 `open` 打开输出文件

### 关键约束

1. **直接替换，不加修订标记**——输出是干净的最终稿
2. **不修改基准 DOCX**——始终输出到新文件
3. **规则冲突检测**：如果两个规则的 find 匹配到同一段落，警告用户
4. **删除操作要谨慎**：replace 为空时清空段落文本即可，不要删除段落 XML 元素（避免破坏文档结构）
5. **JSON 中的 `\n` 换行**：在 docx 中对应插入新段落，不是段内换行
6. **大段替换用直接重建模式**：delete old paras + insert new with style inheritance，不走 find/replace
