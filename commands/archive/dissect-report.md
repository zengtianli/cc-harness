拆解生态流量报告成品，输出结构化 MD 文件，供后续 CC 会话（如 /gen-report、数据准备）使用。

用户提供了参数: $ARGUMENTS

## 输入

`$ARGUMENTS` 为报告文件路径（.md 或 .docx）。

- 如果是 .docx，先在同目录找同名 .md；没有则提示用户先转换
- 如果无参数，在当前 `deliverables/` 下找最新的成品 .md（按文件名日期排序）
- 如果仍找不到，报错退出

## 分析流程

### 1. 读取全文

读取报告 MD 全文。用正则提取所有：
- 章节标题（`# ` / `## ` / `### `）
- 表格标记（`表X.X-X` 开头的行）
- 图片标记（`图X.X-X` 开头的行 或 `![` 开头的行）

### 2. 报告结构

逐章节列出：

```markdown
| 章节 | 标题 | 行号 | 内容性质 |
```

内容性质判断规则：
- 第1章（编制目的与依据）→ 模板（各县通用）
- 第2章 2.1（河湖水系概况）→ 数据（县市基础信息）
- 第2章 2.2（工程排查）→ 数据（DB查询）
- 第3章（生态流量计算）→ 计算（脚本输出）
- 第4章（保障措施）→ 半模板+人工（现场查勘）
- 第5章（结论）→ 人工总结
- 前言 → 半模板（少量县市数据）

### 3. 表格清单

逐表列出以下字段：

| 字段 | 说明 |
|------|------|
| 编号 | 如"表2.1-2" |
| 名称 | 表格标题 |
| 行号 | 在 md 中的起始行 |
| 数据行数 | 不含表头的行数 |
| 字段列表 | 所有列名 |
| 数据来源 | DB表名 / 脚本名 / 外部文件 / 模板 |
| 自动化等级 | ✅可自动 / ⚠️半自动 / ❌人工 / 📋模板 |
| 生成方法 | SQL思路、脚本命令、或"人工摘录" |

**数据来源判断规则：**

| 表格内容特征 | 数据来源 | 自动化 | 生成方法 |
|-------------|----------|--------|---------|
| 水库名录/基本情况（含集雨面积、库容等） | `db_reservoirs.db` → reservoirs | ✅ | SQL 按县市+规模筛选 |
| 供水量统计（含年份×水源类型） | `db_water_stats.db` → water_stats type=supply | ✅ | SQL 按县市+type筛选 |
| 用水量统计（含年份×用途分类） | `db_water_stats.db` → water_stats type=usage | ✅ | SQL 按县市+type筛选 |
| 逐月流量统计（62年×12月） | `db_flows.db` → daily_flows 逐月聚合 | ✅ | SQL GROUP BY year,month + pivot |
| 生态基流计算成果（Tennant+QP） | `eco_flow_calc.py` | ✅ | 脚本计算 |
| 生态最小下泄流量 | 表3.2-3推荐值 × 86400 | ✅ | 上游表推导 |
| 水库排查+未核定目录 | `db_survey.db` → survey_entries | ✅ | SQL 筛选 |
| 水资源总量（不同频率） | 水资源调查评价报告 | ❌ | 人工摘录 |
| 水功能区划分 | 省水功能区划分方案 | ❌ | 人工摘录 |
| 计算方法选择表 | 各县通用 | 📋 | 模板复制 |
| 泄放/监测设施改造建议 | 数据+人工判断 | ⚠️ | 水库名+流量自动填，改造建议人工 |
| 附表（汇总表） | 前面各表拼接 | ⚠️ | 自动拼接 |

对每张✅表，写出具体 SQL 思路（包含表名、WHERE条件、关键字段）。

### 4. 图片清单

逐图列出：

| 编号 | 名称 | 行号 | 来源类型 | 可自动化 |
|------|------|------|---------|---------|

来源类型判断：
- 地理位置/行政区划/水库分布/功能区划 → GIS制图
- 用水结构饼图 → 脚本生成（DB数据）
- 频率曲线 → eco_flow_calc.py 输出
- 水库全景/放水设施 → 现场照片
- 封面logo/签名章 → 模板固定

### 5. 待办事项

自动生成以下 TODO：

**数据缺口：**
- 每张❌人工表 → `- [ ] 人工填写: 表X.X-X {表名}`
- 数据行为空的表 → `- [ ] 数据缺失: 表X.X-X`

**图片缺口：**
- GIS制图类 → `- [ ] GIS制图: 图X.X-X {图名}`
- 现场照片类 → `- [ ] 现场照片: 图X.X-X {图名}`
- 脚本可生成类 → `- [ ] 脚本生成: 图X-X {图名}`

**交叉校验提示：**
- `- [ ] 校验: 表3.3-1流量值 = 表3.2-3推荐成果`
- `- [ ] 校验: 表4.1-1流量值 = 表3.3-1`
- `- [ ] 校验: 附表完整汇总前面各表`
