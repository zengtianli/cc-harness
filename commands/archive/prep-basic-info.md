用户提供了参数: $ARGUMENTS

请执行基础信息数据准备流程。

## 目标

为目标县市生成 6 个基础信息 MD 文件，用于 `/gen-report` 报告生成。

## 流程

### 1. 解析参数

从 `$ARGUMENTS` 解析目标县市名（如 "天台"）。

读取 `~/Work/eco-flow/sync_config.yaml` 获取县市配置（city、stat_name）。

确定目录：
- 目标县市数据目录：`~/Work/eco-flow/{city拼音}-{county}/data/`
- 参考资料存放目录：`~/Work/eco-flow/{city拼音}-{county}/data/参考资料/`

### 2. 读取模板格式

读取景宁的 6 个 MD 文件作为**格式模板**（只参考格式和字段，不抄数据）：
- `~/Work/eco-flow/lishui-景宁/data/01_基本信息.md`
- `~/Work/eco-flow/lishui-景宁/data/02_行政区划.md`
- `~/Work/eco-flow/lishui-景宁/data/03_经济社会.md`
- `~/Work/eco-flow/lishui-景宁/data/04_河流水系.md`
- `~/Work/eco-flow/lishui-景宁/data/05_水文气象.md`
- `~/Work/eco-flow/lishui-景宁/data/06_水资源概况.md`

### 3. 搜索并下载参考资料

逐个搜索以下类型的资料，优先官方来源，优先最新年份（2023/2024）：

**第一轮搜索（水利相关）**：
- `"{stat_name} 水资源公报"` — 最重要，包含供用水、水资源总量
- `"{city} 水资源公报 {year}"` — 市级公报可能包含下辖县区数据
- `"{stat_name} 水利发展统计公报"` — 水利基础设施
- `"{stat_name} 河流水系"` / `"{stat_name} 水文概况"` — 河流信息

**第二轮搜索（基础信息）**：
- `"{stat_name} 统计年鉴"` / `"{stat_name} 国民经济统计公报"` — GDP、人口
- `"{stat_name} 行政区划"` — 乡镇街道
- `"{stat_name} 气候特征"` / `"{stat_name} 气象数据"` — 水文气象
- `"{stat_name} 地理位置 面积"` — 基本信息

**第三轮搜索（补充）**：
- `"{stat_name} 水功能区划"` — 水功能区数据
- `"{stat_name} 水资源调查评价"` — 频率水资源量

对找到的 PDF 和网页内容，用 WebFetch 下载/抓取，保存到 `参考资料/` 目录。

### 4. 逐文件生成

按以下顺序生成 6 个 MD 文件。每个文件：
- 严格按景宁模板的**标题结构和字段**
- 用搜索到的资料填充**实际数据**
- 找不到的字段标记 `[待补充]`
- 数据来源标注为 `<!-- 来源: xxx公报/年鉴, 20xx年 -->`

#### 01_基本信息.md
```
必填字段:
- 地理位置（省市县关系、毗邻县市）
- 地理坐标（经纬度范围）
- 区域面积（总面积，精确到km²）
- 长度宽度（东西/南北跨度）
- 行政中心（县政府驻地）
```

#### 02_行政区划.md
```
必填字段:
- 行政区划总览（街道/镇/乡数量）
- 各乡镇名称列表
- 行政村总数
```

#### 03_经济社会.md
```
必填字段:
- 常住人口（总数、城镇人口、城镇化率）
- GDP（总量、人均）
- 产业结构（三次产业占比）
- 主要产业
```

#### 04_河流水系.md
```
必填字段:
- 水系归属（属于哪个水系/流域）
- 主要河流（名称、长度、流域面积）
- 主要支流
- 水利工程（主要水库）
- 水系特征（河道特征、径流特征）
```

#### 05_水文气象.md
```
必填字段:
- 气候类型
- 年均气温、极端气温
- 年降水量（多年平均、季节分布）
- 日照时数、无霜期
- 灾害性天气
```

#### 06_水资源概况.md
```
必填字段:
- 水资源总量（多年平均、不同频率）
- 供水量（总量、分类占比）
- 用水量（总量、分行业占比）
- 水资源开发利用率
- 水功能区数量和达标率
```

### 5. 质量检查

对每个生成的文件：
1. 检查是否有 `[待补充]` 标记 — 统计缺失字段数
2. 确认数据来源注释完整
3. 确认**没有使用景宁的数据**（grep 排除）

### 6. 完成

报告：
- 6 个文件的生成状态
- 数据完整度（已填 vs 待补充字段数）
- 使用的参考资料列表
- 建议用户核实的关键数据点
- 提示后续步骤：
  1. 人工核实 `[待补充]` 和 `<!-- 待核实 -->` 标记的数据
  2. `/prep-stats` 准备统计数据 Excel
  3. `/prep-engineering` 准备工程排查数据
