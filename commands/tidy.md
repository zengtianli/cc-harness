深度整理当前目录或指定目录。$ARGUMENTS 为目标路径（默认 `.`）。

递归扫描目标目录及其所有子目录（docs/、data/、deliverables/ 等），清理版本冗余、中间产物和垃圾文件。

## 步骤

### 1. 全量扫描

用 `find` 或 `tree` 列出目标目录下**所有文件**（递归），然后逐层分析：

**根目录**：识别散乱文件，应归入 docs/data/deliverables/
**deliverables/**：只保留最终交付版本，删除所有中间版本
**docs/**：清理解包残留、旧版本方案、重复文件
**data/**：一般不动，但清理明显的临时文件

### 2. 识别问题

逐个目录输出问题清单：

#### 垃圾文件（直接删除，不用问）
- Office 临时文件：`~$*`、`.~lock.*`
- macOS 系统文件：`.DS_Store`、`Thumbs.db`
- docx 解包残留：包含 `word/`、`_rels/`、`[Content_Types].xml` 的目录（这是手动解压 .docx 产生的）
- hash 命名文件：如 `a354070ccd654ed08b27245268f0e84b.png`
- 转换副产品：同一文档的 `.rtf`、`.txt` 版本（当 .docx 和 .pdf 都在时）

#### 版本链（表格展示，等确认）

识别同一文档的版本演进链，按时间排序：

```
文档名（去掉日期和后缀）→ 列出所有版本 → 标记最终版 → 其余建议删除
```

判断规则：
- 日期越大 = 越新（1126 < 0112 < 0123 < 0128 < 0310 < 0313）
- `成品`/`final`/`定稿` = 最终版
- `_ztl`/`_dh`/`_reviewed`/`_cleaned`/`_fixed`/`校核后` = 中间审核版
- `-1`/`-2`/`2` 结尾 = 小修版本
- 最终版的 `.docx` + `.pdf` 都保留
- 最终版的 `.md` 如果是从 docx 导出的工具产物（不是手写的），可删

#### 散乱文件（建议归位）
不在正确子目录中的文件建议移动：
- 合同、审查意见、会议记录 → docs/
- 方案最终版 → deliverables/
- 数据表格 → data/
- 发票 → docs/

### 3. 输出整理方案

**按目录分组**，每组一个表格：

```markdown
## 根目录 (/)
| 文件 | 判断 | 操作 |
|------|------|------|
| ... | ... | ... |

## deliverables/
| 文件 | 判断 | 操作 |
|------|------|------|
| 方案1126.docx | 旧版本 | 🗑 删除 |
| 方案0313成品.docx | ✅ 最终版 | 保留 |
| 方案0313成品.pdf | ✅ 最终版 | 保留 |
| quarto_*/ | 构建项目 | 保留 |

## docs/
| 文件 | 判断 | 操作 |
|------|------|------|
| 0景宁...0123/ (word解包) | 垃圾 | 🗑 删除整个目录 |
| 方案0123.docx | 旧版本 | 🗑 删除 |
| 合同.docx | 合同文件 | 保留 |
```

末尾附一行汇总：`共 N 个文件，保留 X，删除 Y，移动 Z`

### 4. 等用户确认

**不要自动执行。** 说"展示完毕，确认后执行"。用户可以逐条修改。

### 5. 执行

按确认的方案操作。完成后 `tree` 展示最终结构。

## 不要动的文件
- `_project.yaml`、`_meta.yaml`、`_index.md`
- `README.md`
- `.claude/` 目录
- data/ 下的核心数据文件（xlsx、shp 等）

## 遇到不确定的
标注 `⚠ 需确认` 并说明原因，不要猜。
